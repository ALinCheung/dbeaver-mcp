# Primary Keys

Oracle 数据库的表是堆表（heap organized），主键唯一标识每行，但不像 MySQL InnoDB 那样聚簇存储。

## IDENTITY（Oracle 12c+）

推荐使用身份列，与 SQL 标准兼容：

```sql
CREATE TABLE users (
  id NUMBER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  name VARCHAR2(100) NOT NULL,
  email VARCHAR2(255) NOT NULL
);

-- 或使用 BY DEFAULT（允许显式插入）
CREATE TABLE users (
  id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name VARCHAR2(100) NOT NULL
);
```

## 序列（Sequence）

Oracle 传统方式使用序列：

```sql
CREATE SEQUENCE users_seq START WITH 1 INCREMENT BY 1;

CREATE TABLE users (
  id NUMBER PRIMARY KEY,
  name VARCHAR2(100) NOT NULL
);

INSERT INTO users (id, name) VALUES (users_seq.NEXTVAL, 'John');

-- 获取当前值
SELECT users_seq.CURRVAL FROM DUAL;
```

## 复合主键

```sql
CREATE TABLE user_roles (
  user_id NUMBER NOT NULL,
  role_id NUMBER NOT NULL,
  PRIMARY KEY (user_id, role_id)
);
```

## GUID/UUID

Oracle 提供了 `SYS_GUID()` 函数生成 32 字节的 GUID：

```sql
CREATE TABLE objects (
  id RAW(16) DEFAULT sys_guid() PRIMARY KEY,
  name VARCHAR2(100)
);

-- 或者使用 DBMS_CRYPTO.GENERATE_BYTES
```

## 最佳实践

- 优先使用 IDENTITY（Oracle 12c+）或序列
- 避免使用字符串作为主键
- 复合主键应保持最小化
- 考虑使用全局临时表（Global Temporary Table）进行中间计算

## 主键约束和索引

```sql
-- 主键自动创建唯一索引
-- 查看主键索引
SELECT index_name, table_name FROM user_indexes WHERE uniqueness = 'UNIQUE';

-- 自定义主键索引存储
CREATE TABLE users (
  id NUMBER,
  name VARCHAR2(100),
  CONSTRAINT pk_users PRIMARY KEY (id) USING INDEX (
    CREATE INDEX pk_users_idx ON users(id) TABLESPACE users_idx
  )
);
```

## 注意事项

- NUMBER 类型最大精度为 38 位数字
- 主键索引默认 tablespace 与表相同
- 删除表时主键约束和序列不会自动删除
